Unicode-Kodierung auf eihem Win32 Server

Die Datenbank ist mit der Unicode-Kodierung (UTF8) zur Speicherung der Daten erzeugt und wird mit einem PostgreSQL 8.0 Server auf einem Win32 Server betrieben.

Die PostgreSQL 8.0 Win32 Implementierung kann mit Unicode Zeichenketten nicht anhand der korrekten Collation Sequence sortieren, da das Betriebssystem diese Funktion nicht bietet.

Für diese Installation wird empfohlen, eine lokal spezifische Kodierung zu verwenden, den Cluster auf einem anderen Betriebssystem laufen zu lassen oder das Datenbanksystem auf PostgreSQL 8.1 oder folgende zu aktualisieren, da spätere Versionen zusätzlichen Code enthalten um die Schwäche von Win32 auszugleichen.
